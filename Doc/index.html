VR Base Package 강의 문서

처음 프로젝트 열기 → 안드로이드(Quest) 전환 → 설치/설정 → 데모에서 박스 잡기
Unity 6 (6000.4.8f1) · OpenXR · URP · 핸드포즈 그립

이 문서는 빈 Unity 프로젝트에서 vr-base-package를 설치하고, 안드로이드(Meta Quest) 플랫폼으로 전환해 데모 씬에서 왼쪽 컨트롤러로 박스를 잡는 것까지 따라 하는 강의용 가이드입니다. 순서대로 진행하세요.

목차

  1. 사전 준비물
  2. 프로젝트 처음 열기
  3. 플랫폼을 Android로 변경
  4. 패키지 추가 (Git URL)
  5. 필수 패키지 확인
  6. 필수 샘플 임포트
  7. 프로젝트 설정 (XR / OpenXR / URP / Input)
  8. Project Validation (자동 점검·수정)
  9. 데모 실행 — 왼쪽 컨트롤러로 박스 잡기
  10. 문제 해결

0. 사전 준비물

  • Unity 6 (권장 6000.4.8f1) — Unity Hub 설치 시 Android Build Support(OpenJDK, Android SDK & NDK 포함) 모듈 체크
  • Meta Quest 기기 (Quest 2/3/Pro) + USB 케이블(Quest Link) 또는 무선(Air Link)
  • PC 미리보기용 Meta Quest Link 앱
핵심핸드트래킹 포즈는 실제 기기(Quest/Link)에서만 정확히 동작합니다. 에디터 단독 재생으로는 손 추적이 보이지 않습니다.

1. 프로젝트 처음 열기

  1. Unity Hub ▸ New project → 템플릿 Universal 3D (URP) 선택 → 생성
    (이미 받은 프로젝트가 있으면 Open 으로 해당 폴더를 엽니다.)
  2. 에디터가 열리면 Project/Hierarchy 창이 보이는지 확인합니다.
참고URP 템플릿으로 시작하면 렌더 파이프라인(URP)이 자동 지정되어 6단계의 URP 설정을 건너뛸 수 있습니다.

2. 플랫폼을 Android로 변경

Quest는 안드로이드 기반입니다. 가장 먼저 빌드 타깃을 Android로 바꿔야 이후 설정(OpenXR Android 탭 등)이 올바르게 적용됩니다.

  1. 메뉴 File ▸ Build Profiles 열기 (Unity 6 기준. 구버전은 Build Settings)
  2. 플랫폼 목록에서 Android 선택
  3. Switch Platform 클릭 → 에셋 재임포트가 끝날 때까지 대기
  4. 전환되면 Android 옆에 활성 표시(Unity 아이콘)가 붙습니다.
시간이 걸립니다최초 플랫폼 전환은 모든 에셋을 다시 임포트하므로 수 분 걸릴 수 있습니다. 멈춘 게 아니니 기다리세요.

3. 패키지 추가 (Git URL)

  1. 메뉴 Window ▸ Package Manager
  2. 좌측 상단 + ▸ Add package from git URL...
  3. 아래 주소 입력 후 Add: 
    https://github.com/smilejsu82/vr-base-package.git
버전 고정특정 버전을 쓰려면 URL 뒤에 태그를 붙입니다: ...vr-base-package.git#v1.0.0
자동 설치이 패키지는 완전 독립 패키지라 핸드포즈 스크립트·셰이더·도구가 모두 들어 있습니다. 추가하면 아래 XR 의존성도 자동 설치됩니다.

4. 필수 패키지 확인

3단계에서 자동 설치됩니다. 자동 설치가 안 되면 Package Manager ▸ + ▸ Add package by name... 로 식별자/버전을 직접 입력하세요.

5. 필수 샘플 임포트

데모 씬의 XR 리그·손 모델은 아래 두 패키지의 샘플을 사용합니다(라이선스/용량상 패키지 미포함). Window ▸ Package Manager 에서 각 패키지를 선택 → Samples 탭 ▸ Import.

패키지임포트할 샘플
XR Hands (com.unity.xr.hands)HandVisualizer
XR Interaction Toolkit (com.unity.xr.interaction.toolkit)Starter Assets

임포트하면 다음 위치에 들어옵니다:

Assets/Samples/XR Hands/1.7.3/HandVisualizer
Assets/Samples/XR Interaction Toolkit/3.4.1/Starter Assets

6. 프로젝트 설정 (XR / OpenXR / URP / Input)

6-1. XR Plug-in Management

Project Settings ▸ XR Plug-in Management 에서 PC(데스크톱) 탭과 Android 탭 양쪽 모두:

  • ☑ Initialize XR on Startup
  • Plug-in Providers ▸ ☑ OpenXR

6-2. OpenXR

Project Settings ▸ XR Plug-in Management ▸ OpenXR — PC/Android 탭 공통:

  • Enabled Interaction Profiles ▸ + → Oculus Touch Controller Profile 추가
  • OpenXR Feature Groups 체크: ☑ Hand Interaction Poses, ☑ Hand Tracking Subsystem

탭별 권장 값:

항목PC(데스크톱) 탭Android(Quest) 탭
Render ModeSingle Pass InstancedSingle Pass Instanced \ Multi-view
Auto Color Submission Mode☑☑
Latency OptimizationPrioritize RenderingPrioritize Rendering
Offscreen Rendering Only (Vulkan)—☑
OpenXR 경고(▲) 아이콘위 Interaction Profile / Hand 기능이 빠지면 표시됩니다. 둘을 추가하면 사라집니다.

6-3. 렌더 파이프라인 (URP)

  • Project Settings ▸ Graphics ▸ 활성 렌더 파이프라인을 URP 에셋으로 지정 (URP 템플릿이면 자동)
  • Project Settings ▸ Quality 의 각 레벨도 URP 에셋 사용 확인

6-4. 입력 시스템

  • Project Settings ▸ Player ▸ Active Input Handling = Input System Package (New) (또는 Both)

6-5. (Quest 빌드 시) Android Player 설정

  • Player ▸ Other Settings: Scripting Backend = IL2CPP, Target Architectures = ARM64
  • Minimum API Level은 사용 기기/스토어 요구사항에 맞춰 지정

7. Project Validation (자동 점검·수정)

Project Settings ▸ XR Plug-in Management ▸ Project Validation 에서 설정이 올바른지 자동 점검합니다. PC 탭과 Android 탭 양쪽 모두 확인하세요.

  1. 우측 상단 Fix All 클릭 → 자동 처리 항목:
    • [OpenXR] PoseControl → InputSystem.XR.PoseControl 로 전환
    • [OpenXR] Vector2Control → StickControl thumbstick 으로 전환
    • [XR Plug-in Management] Run In Background 활성화
    • [XRI][Starter Assets] Interaction Layer 31 = Teleport 설정
  2. 남는 빨간 오류 “TextMesh Pro - TMP Essentials must be installed” → 오른쪽 Edit 클릭 → TMP Importer 창에서 Import TMP Essentials
    (또는 Window ▸ TextMesh Pro ▸ Import TMP Essential Resources)
  3. 모든 이슈가 사라지면 설정 완료입니다.
순서 주의Fix All은 5단계의 샘플(Starter Assets/HandVisualizer)을 임포트한 뒤에 실행해야 관련 검사가 모두 나타납니다.

8. 데모 실행 — 왼쪽 컨트롤러로 박스 잡기

  1. Demo 샘플 Import: Package Manager ▸ VR Base Package ▸ Samples ▸ Demo ▸ Import
  2. 임포트된 데모 씬 열기: 
    Assets/Samples/VR Base Package/1.0.0/Demo/Test_1.unity
  3. Hierarchy 구성 확인: 
    Test_1
├─ Directional Light
├─ XR Origin (VR)
│  └─ Camera Offset
│     ├─ Main Camera
│     └─ Left Controller
│        ├─ Visuals
│        ├─ Left_NearFarInteractor   ← 잡기용 인터랙터
│        └─ Hand Visualizer
└─ Cube                              ← 잡을 박스(그랩 대상)
  4. 헤드셋 연결(Quest Link/무선) 후 에디터 상단 ▶ Play
  5. 왼손 컨트롤러를 Cube에 가져가 Grip 버튼으로 잡으면, 핸드포즈가 적용되어 손이 박스 그립 모양으로 잡힙니다.
성공 기준왼손이 큐브를 잡는 순간 손가락이 미리 녹화된 그립 포즈로 자연스럽게 감기면 성공입니다.

9. 문제 해결

증상점검 / 해결
손이 그립 포즈로 안 잡힘Cube에 그립 스크립트 + HandPoseData가 할당돼 있는지, Left_NearFarInteractor가 활성인지 확인
컨트롤러/손이 안 보임OpenXR에 Oculus Touch Controller Profile + Hand Tracking Subsystem이 켜져 있는지, 헤드셋 연결 확인
UI 텍스트가 깨짐 / 핑크TMP Essentials 미설치 — 7단계의 TMP Import 수행
화면이 분홍색(머티리얼 깨짐)URP가 활성 렌더 파이프라인으로 지정됐는지(6-3) 확인
입력이 전혀 안 먹힘Active Input Handling = New(또는 Both) 확인 후 에디터 재시작
OpenXR 옆 ▲ 경고Interaction Profile / Hand 기능 누락 — 6-2 재확인
중요이 패키지는 com.smilejsu82.vr-handpose-fire의 기능을 포함한 독립 버전입니다. 두 패키지를 동시에 설치하지 마세요 — 스크립트 GUID·asmdef가 겹쳐 컴파일 오류가 납니다.
